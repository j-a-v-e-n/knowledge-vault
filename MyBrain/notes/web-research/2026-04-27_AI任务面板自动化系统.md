# AI 任务面板自动化系统（思瑶视频）

> 用一个看板文件 + 事件驱动调度 + 多 agent 执行 + 人机阻塞授权机制，让 Claude Code 在你睡觉时也能继续推进任务——以"你不在电脑前 AI 还能不能干活"作为区分真假自动化的唯一标准。

---

## 解决了什么问题

普通人用 AI 的常见模式是"人工开关"：打开窗口 → 给指令 → 等结果 → 关窗口。这种模式下 AI 像一个**只在你盯着才肯做饭的保姆**——窗口一关就停摆，多个任务无法并行，半夜的空闲时间被完全浪费。

作者提出真假自动化的判别标准：**你去睡觉时 AI 还能不能继续干活**。如果不能，那就还是手动遥控，不是自动化。

## 核心结论：五层架构

整个系统从用户最近到最远分五层。每一层有一个生活化类比：

| 层 | 角色 | 类比 | 实现 |
|---|---|---|---|
| 任务面板（接口层） | 人和 AI 共用的可读写文件，记录所有任务 | 家里的小黑板 | 一个 Markdown 看板文件 |
| 调度层 | 决定何时唤醒 AI | 门铃 + 闹钟 | `fswatch`（事件驱动）+ `launchd`（定时兜底）|
| 执行层 | 真正干活的 agent | 工头 + 工人 | 主 agent 调度 + 子 agent 执行子任务 |
| 业务逻辑层 | AI 的自我行为规则 | 行为守则 | 写在 Claude skill 里的判断/路由/优先级规则 |
| 状态与日志层 | 全部改动按时间写入 | 餐厅监控录像 | 结构化日志文件，可回溯 |

类比让架构容易记：你在小黑板上写一笔，门铃响了，工头听到了就喊工人来，工人按守则干活，监控录像把每一步都录下来。

## 关键设计

### 1. 事件驱动 > 定时轮询

作者最强调的一个工程经验：**先做的版本是每 30 分钟定时轮询，但很费 token**。原因是大部分轮询时面板根本没变化，AI 上去溜达一圈又回来，纯空转。

后来改成**事件驱动**：只有当用户真的修改了看板文件，`fswatch` 才会触发 AI 唤醒。`launchd` 仍然保留作为定时兜底（每小时一次），防止 `fswatch` 丢事件。

> 这里的设计模式是"事件驱动主+定时轮询副"——主路径靠事件即时响应、省 token；副路径靠定时确保最终一致性、防漏。

### 2. "blocked on 用户授权" 机制

AI 遇到自己无法决定的事时，**不会瞎猜也不会硬推**。它会在该子任务上写一句类似 `blocked on 思瑶授权` 的标记，然后停下，转去做别的可推进的任务。

这避免了两种典型失控：
- **AI 越权**——擅自做需要用户拍板的决定
- **AI 偷懒**——卡住一项就停掉所有项目

视频中具体例子：portfolio 网站的子任务 a–k，里面有 Claude 可以直接动手的（修文件、补 bug），也有需要用户拍板的（语言风格、文案方向）。需要用户决策的就丢回看板等用户处理。

### 3. 主 agent + 子 agent 派发

主 agent 把控全局任务流。需要"上网搜个东西"这种局部子任务时，主 agent 会**派一个子 agent** 去执行。工头喊工人，工人干完回来交活。

### 4. 业务逻辑写成 skill

业务逻辑层是用户最能定制的部分。视频里展示的 Claude skill 包含：

- **任务复杂度判定**：哪些自己做、哪些停下问用户
- **失败原因分类**：任务做不下去时标注是哪类阻塞、并给出建议处理方式
- **路由表**：谁负责什么事
- **优先级标签**：P0/P1/P2 + 业务线归属
- **注意力分配规则**：低优先级任务不能占用高优先级时间，低优先级有限时
- **个性化适配**：作者特别提到自己有 ADHD 和启动困难，所以 skill 里写了"分配给用户的任务必须具体、单次可完成、要说清楚为什么"

> 注意力分配规则这条很关键——它不是技术问题，是把"用户认知特性"显式编码进 AI 行为，让 AI 真正适配人，而不是反过来。

### 5. 全程可回溯

每次改动按时间写入结构化日志。作者类比为"餐厅的监控录像"——保证每一步都可查、谁在什么时候做了什么改动一目了然。

## 意味着什么

**对个人生产力的改变：**
- 下班和睡觉时间从"AI 停摆"变成"AI 继续工作"
- 多任务可同时并行（看板里多条并跑）
- 拖延症问题被显著缓解——任务一旦丢进看板，能做的部分立刻开始做
- 作者自评效率提升 5–10 倍

**26 秒模拟运行：** 视频展示了一个模拟跑——全程 26 秒、用户没开任何窗口，AI 把一系列任务跑完。

## 边界与局限（视频自己澄清的 FAQ）

| 问题 | 回答 |
|---|---|
| AI 会不会偷偷做你没同意的事？ | 只做写进看板的事 |
| 关电脑后怎么继续跑？ | 后台服务运行，不依赖打开的窗口 |
| 要钱吗？ | 系统本身不要钱，但调用 AI 消耗 Claude Pro/Max 订阅额度（视频中口误为"GPS"）|
| 怎么关掉？ | 在 terminal 写一行命令终止全部服务（视频中口误为"弹匣"）|
| 别人能看你的数据吗？ | 不能，全部本地运行 |
| 行为可以改吗？ | 可以，skill 文件随时编辑 |
