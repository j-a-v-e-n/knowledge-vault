### Title
**UniStory: A Unified Autoregressive–Diffusion Transformer for Visual-Textual Story Generation**

### Abstract (optional, 3-4 sentences)

We propose **UniStory**, a unified deep generative model that jointly produces a long-form narrative and a representative illustration from a short textual prompt. UniStory adopts a single Transformer backbone that simultaneously performs next-token prediction on text tokens and denoising diffusion on continuous image latents, following the Transfusion paradigm. A lightweight two-stage text decoder first generates a compact narrative sketch, then expands it into the full story, while a CLIP-based alignment objective ensures the generated image faithfully reflects the story's content.

---

### 1. Introduction

Given a short prompt $c$ (e.g., *"We build a Disney park on the moon"*), our goal is to model the joint distribution $p_\theta(s, I \mid c)$, where $s = (s_1, \dots, s_N)$ is a long narrative and $I \in \mathbb{R}^{H \times W \times 3}$ is an illustrative image. Two key challenges arise:

1. **Heterogeneous modalities.** Text is discrete and sequential; images are continuous and spatial. A single model must accommodate both.
2. **Semantic coherence.** The image must visually reflect the story it accompanies — not just the prompt.

We address (1) by unifying autoregressive (AR) text modeling and diffusion-based image modeling inside one Transformer, and (2) by conditioning image diffusion on the *generated story* (not only the input prompt) and adding a CLIP-based alignment loss.

---

### 2. Model Architecture

**Figure 1** illustrates the UniStory pipeline.

```
           Short Prompt c
                 │
        ┌────────▼────────┐
        │  Text Tokenizer │
        └────────┬────────┘
                 │ text tokens
   ┌─────────────▼─────────────────────────────┐
   │       Unified Transformer (L layers)      │
   │    shared self-attention over text +      │
   │    image patch tokens; causal mask on     │
   │    text, bidirectional within image block │
   └───────┬──────────────────────┬────────────┘
           │ text                 │ image
   ┌───────▼────────┐   ┌─────────▼──────────┐
   │  Text Head     │   │  Diffusion Head    │
   │  (AR, softmax) │   │  (ε-prediction)    │
   └───────┬────────┘   └─────────┬──────────┘
           │                      │
   Long Story s           Image Latent z₀
                                  │
                         ┌────────▼────────┐
                         │  VAE Decoder D  │
                         └────────┬────────┘
                                  │
                              Image I
```

**Components:**

- **Text path.** Tokenizer (BPE) → Transformer → linear head → softmax over vocabulary.
- **Image path.** A pretrained VAE encoder $E$ maps the image to a low-dimensional latent $z_0 = E(I) \in \mathbb{R}^{h \times w \times d}$. The same Transformer operates on flattened latent patches, with a diffusion head predicting the noise $\epsilon$.
- **Modality-specific adapters.** Small MLPs project text embeddings and image patch embeddings into a shared $d$-dimensional space at the input, and project back at the output.
- **Attention mask.** Causal (left-to-right) across text positions; **bidirectional within the image block** (image patches attend to each other freely, but respect the text→image causal order globally).
- **Two-stage narrative decoding (sketch-then-expand).** To improve long-story coherence, we first autoregressively decode a short *narrative sketch* $\tilde{s}$ (≤ 50 tokens), then use it as a prefix to decode the full story $s$ (~500 tokens). This is inspired by DiffuStory's latent-plan approach.

### 3. Training Objectives

Let $x = (c, s, z_0)$ denote one training example (prompt, story, image latent).

**3.1 Autoregressive text loss** (course L1b — LLM as chain BN):

$$
\mathcal{L}_{\text{LM}}(\theta)
= -\mathbb{E}_{(c,s)} \left[\sum_{t=1}^{N} \log p_\theta\bigl(s_t \mid s_{<t},\, c\bigr)\right].
$$

**3.2 Diffusion loss for image latents** (DDPM objective, equivalent to an ELBO — course L3):

The forward noising process is
$$
q(z_t \mid z_0) = \mathcal{N}\bigl(z_t;\, \sqrt{\bar{\alpha}_t}\, z_0,\, (1-\bar{\alpha}_t)\mathbf{I}\bigr).
$$
The network $\epsilon_\theta$ predicts the added noise, conditioned on the prompt $c$ **and** the generated story $s$:
$$
\mathcal{L}_{\text{diff}}(\theta)
= \mathbb{E}_{z_0,\,\epsilon,\,t}
\Bigl[\,\bigl\|\, \epsilon - \epsilon_\theta(z_t,\, t,\, c,\, s) \,\bigr\|_2^2\,\Bigr],
\quad \epsilon \sim \mathcal{N}(0,\mathbf{I}),\ t \sim \mathcal{U}\{1,\dots,T\}.
$$

**3.3 Cross-modal alignment loss.** To ensure the image reflects the *story*, we use a pretrained CLIP model:
$$
\mathcal{L}_{\text{align}}(\theta)
= 1 - \cos\!\bigl(\phi_{\text{txt}}(s),\; \phi_{\text{img}}(\hat{I})\bigr),
$$
where $\hat{I} = D(\hat{z}_0)$ is the one-step-reconstructed image and $\phi_{\text{txt}}, \phi_{\text{img}}$ are frozen CLIP encoders.

**3.4 Total objective:**

$$
\boxed{\;\mathcal{L}_{\text{UniStory}}(\theta) \;=\; \mathcal{L}_{\text{LM}} \;+\; \lambda_{\text{diff}}\,\mathcal{L}_{\text{diff}} \;+\; \lambda_{\text{align}}\,\mathcal{L}_{\text{align}}\;}
$$

with $\lambda_{\text{diff}} = 1.0$, $\lambda_{\text{align}} = 0.1$ as default weights.

---

### 4. Training Strategy

**Stage 1 — Large-scale interleaved pretraining.** Train on public paired corpora (LAION-5B for image-caption, Books3/The Pile for text), using the mixed objective above. Text-only and image-only batches use only the corresponding loss term.

**Stage 2 — Story-image fine-tuning.** Fine-tune on triples $(c,\, s,\, I)$ — short prompt, long story, representative image — constructed from WritingPrompts (Reddit) augmented with illustrations generated by a teacher model and filtered by CLIP score.

**Optimization.** AdamW, cosine LR schedule, warmup 2k steps, classifier-free guidance (CFG) dropout of $c, s$ at 10% probability to enable guided sampling at inference.

**Stabilization tricks.** We apply QK-Norm (Chameleon) and z-loss to prevent logit explosion in mixed-modal training.

---

### 5. Inference

Given prompt $c$:

1. **Sketch generation.** Sample $\tilde{s} \sim p_\theta(\cdot \mid c)$ autoregressively, $\leq 50$ tokens.
2. **Story expansion.** Sample $s \sim p_\theta(\cdot \mid c, \tilde{s})$ with nucleus sampling ($p=0.9$).
3. **Image sampling.** Starting from $z_T \sim \mathcal{N}(0, \mathbf{I})$, run $T$ reverse diffusion steps with CFG:
$$
\hat{\epsilon} = (1+w)\,\epsilon_\theta(z_t, t, c, s) - w\,\epsilon_\theta(z_t, t, \varnothing),
$$
with guidance scale $w=3.0$. Decode $I = D(z_0)$.

---

### 6. Discussion

**Why hybrid AR+diffusion?** Pure autoregressive image generation (Chameleon) requires VQ tokenization, which bottlenecks image fidelity. Pure diffusion models cannot natively generate long coherent text. Transfusion-style hybridization retains each paradigm's strengths. **Why condition image on story, not prompt?** The prompt underspecifies the scene; the story determines concrete visual elements (e.g., *"Space Princess Castle"* emerges only after generation). **Limitations.** Serial inference (story then image) prevents full parallelism; we leave joint AR-diffusion sampling for future work.
