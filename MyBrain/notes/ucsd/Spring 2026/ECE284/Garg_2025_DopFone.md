# DopFone: Doppler-based Fetal Heart Rate Estimation Using Commodity Smartphones

> 把任意智能手机变成胎心率传感器——扬声器发射 18 kHz 超声，麦克风接收腹部反射波，AdaBoost 回归器结合声学特征 + 母体年龄/BMI/孕周得到 FHR；在 23 名孕妇上 MAE 2.1±1.3 BPM、95% LoA ±4.9 BPM，完全在临床可接受 ±8 BPM 内。

---

## 这项研究在解决什么问题

胎心率（FHR）是产前护理的核心指标，正常 110–160 BPM，偏离即提示胎儿窘迫。临床用的多普勒超声 1–3 MHz、专业培训才能用、设备贵 $500–$2000；家用方案要么是 $145 的低端手持多普勒（精度未临床验证），要么是穿戴腰带（不舒服、电池焦虑），要么是手机应用商店里没有任何临床验证的小程序。**全球每年 200 万死胎**，其中相当部分发生在低中收入国家——那些地方既缺设备也缺训练有素的 OB/GYN。

DopFone 的赌注是：**全球已有 50 亿台智能手机，能不能用它们自带的扬声器和麦克风做出临床级 FHR 监测？** 关键的物理障碍是：临床多普勒在 1–3 MHz 工作（细胞级运动分辨），而智能手机扬声器**最高只能干净发射 ≤20 kHz**——频率低 100 倍意味着信号穿透腹部组织后微弱很多，胎儿小心脏的本就微弱搏动信号要在母体呼吸 / 蠕动 / 胎动等噪声里捞出来。

## 核心结论：18 kHz 多普勒 + AdaBoost 回归 = 临床级精度

|方案 | MAE (BPM) | 95% LoA | r | 是否过临床阈值 (±8 BPM) |
|------|-----------|---------|---|--------------------------|
| **ML @ 18 kHz** | **2.12** | **[-4.90, +4.86]** | **0.89** | ✅ |
| ML @ 20 kHz | 2.57 | [-2.93, +8.07] | — | ⚠️ 上界出界 |
| 信号处理 @ 18 kHz | 4.77 | [-2.32, +11.86] | — | ❌ |
| 信号处理 @ 20 kHz | 6.69 | [-3.62, +17.01] | — | ❌ |

90% 的预测误差 < 3.3 BPM。Bland-Altman 偏差 -0.02 BPM，几乎无系统偏差。Pearson r = 0.89, p < 0.001。

跨 9 种回归器对照（DT / Lasso / KNN / ElasticNet / Ridge / 线性回归 / RF / GB / SVR），**AdaBoost** 全面胜出。

## 为什么会这样

### 类比开路：不是"瞄准心脏"而是"听整面墙"

> **类比**：临床多普勒像狙击手——要正对胎心、对角度、涂耦合凝胶，瞄准微小运动。DopFone 更像把**听诊器贴到墙上听整间屋子的振动**——不需要对准心脏，因为整个母腹表面都在跟着胎心搏动微振，方法捕获的是整个腹腔表面的"振动剖面"，而非单点位移。

这个机制解释了 DopFone 的两个核心鲁棒性发现：（1）手机位置不重要，（2）BMI 高的孕妇虽然信号衰减但仍可用。

### 物理：多普勒频移
$$ \Delta f = \frac{2v\cos\theta}{c} f_0 $$

胎心搏动让母腹表面以 ~1.8 Hz（FHR ≈ 110 BPM 时）周期性微振，载波 18 kHz 反射回来时被调制——频谱里就藏着 FHR。

### 工程权衡：为什么是 18 kHz 不是 20 kHz

智能手机扬声器为可听音频设计，**抗混叠滤波器从 20 kHz 开始急剧衰减**（rolloff），20 kHz 输出能量已经显著低于 18 kHz；同时 18 kHz 波长更长，**组织穿透更好**（牺牲分辨率换穿透）。这一权衡在 Fig 7 的跨手机频响测试里得到验证：

![[Garg_2025_DopFone_page18_Fig7_cross_phones.png]]

> Fig 7：(a) iPhone 13/15/16 Pro Max + Galaxy A15 + Pixel 7 在通过明胶模型测得的 15–22 kHz 频响——**18–20 kHz 区间四款手机响应几乎平直，21 kHz 之后能量陡降**。这一图说明 18 kHz 不是 iPhone 独有的工程偶然，而是**主流智能手机硬件共同的稳定区间**——为跨设备部署铺平道路。(b) 8 种手机/胖瘦组合两两的相关系数全部 > 0.85。(c) 同款手机的"普通 vs 肥胖"明胶模型频响仅差约 4 dB——和真实参与者中正常 vs 肥胖 BMI 的 MAE 差异（1.8 vs 2.4 BPM）数量级一致。

## 怎么证明的：23 名孕妇 + 4 种特征域 + LOOCV

### 实验设计

**参与者**：23 名孕妇，孕中/孕后期，年龄 32（25–40），孕周 31（19–39），BMI 33（23–67），ground truth FHR 142（129–155）BPM。**有意招募了 BMI 偏高的样本**——这是高并发症风险人群，对低资源场景的代表性更强。

**Ground truth**：Summit Doppler LifeDop 250（3 MHz 临床多普勒，自身误差 ±3 BPM）由医生定位最佳点测一分钟逐拍平均。

**手机记录**：iPhone 15，自定义 iOS app，扬声器连续发射 18 kHz 正弦波，麦克风以 48 kHz 采样录音 60 秒/次；每位参与者在 3 个位置 × 多个角度共 4–6 次录音，**外加 1 次"用户自选位置"实验**。

![[Garg_2025_DopFone_page08_Fig1_phone_placement.png]]

> Fig 1：手机底部贴皮肤，扬声器和麦克风对着腹部。(B) 3 个测试位置——医生定位的"最佳"点 + 对面（最远）+ 肚脐（普通用户最容易找的解剖标志）。(C) 三个测试角度（60° / 90° / 120°）。这一图所体现的实验结构本身就是论点：作者**主动测试**手机放歪、放错位置、放偏角度时方法是否还可用——为后续"位置无关性"的鲁棒性论证准备数据。

### 信号处理 baseline

![[Garg_2025_DopFone_page09_Fig2_SP_pipeline.png]]

> Fig 2：传统 SP 流水线 = 4 阶 Butterworth 带通（18 kHz±1 kHz）→ spectrogram（256 窗 / 64 跳 / Blackman 窗）→ **频率重新分配**算法锐化频谱→ novelty function（相邻帧差和，提取频率突变）→ 高斯平滑 → peak detection → FHR = 60/Δt。这条流水线**只达到 MAE 4.77 BPM**——已经能用但 95% 上界 11.86 BPM 超出 ±8 BPM 临床阈值。

### ML pipeline

![[Garg_2025_DopFone_page10_Fig3_ML_features.png]]

> Fig 3：4 种特征域同时喂回归模型——（1）**时域**：Estimated Tempo / RMS Energy 均值 / Peak Density / Upper Half Energy；（2）**频域**：PLP Tempo（基于 librosa 的节拍跟踪 + 110–160 BPM 生理先验）/ Spectral Centroid Mean；（3）**倒谱**：8 个 MFCC 均值 + 前 4 个 MFCC 标准差；（4）**人口学**：孕周 / 母体 BMI / 母龄。这一图所体现的关键设计是：**人口学特征作为"上下文"调整声学特征的解读**——同一信号强度在 BMI 28 和 BMI 50 的孕妇身上意味着不同的真实信噪比。

### 关键结果

![[Garg_2025_DopFone_page12_Fig4_CDF.png]]

> Fig 4：4 方案的累积误差分布——**蓝实线（ML@18kHz）爬升最陡**，意味着大部分预测都集中在小误差区。其他三条线尤其紫色虚线（SP@20kHz）拖得很长——说明 20 kHz 在手机硬件上**确实接近无效**。Table 4：ML@18kHz 的 MAE 2.12 BPM、95th 百分位 4.38 BPM——**95% 病例的误差都在 4.4 BPM 以内**。

![[Garg_2025_DopFone_page13_Fig5_BlandAltman.png]]

> Fig 5：(a) Bland-Altman 图——红线是 95% LoA [-4.90, +4.86] BPM，**绿色虚线是临床可接受 ±8 BPM 阈值**——红线完全在绿线之内。这就是"临床级"的视觉证据。(b) 相关图——AdaBoost 预测与 ground truth 散点紧贴对角线（r = 0.89）。

## 鲁棒性：在哪些条件下 DopFone 还能跑

![[Garg_2025_DopFone_page16_Fig6_confounds.png]]

> Fig 6：4 种可能影响精度的因素——**孕周** (a)：28–31w 最优（1.6 BPM），孕早期 < 27w 略高（2.0 BPM，胎心小且位置乱），36+ 周略高（2.4 BPM，胎位拥挤、母腹增厚）。**BMI** (b)：< 30 是 1.8 BPM，≥ 30 是 2.4 BPM——33% 升幅，源自 18 kHz 信号被脂肪组织衰减。**手机位置** (c)：医生最佳点 1.9 BPM vs 离最佳点最远的位置 2.3 BPM——差异不显著，**位置无关性**。**角度** (d)：60° 1.4 BPM 最优、90° 1.8、120° 1.6——意外的是 90°（直角）最差，可能源于反射界面驻波。

**用户自由放置实验**：单独让 7 位参与者自己选位置和角度（无医生指导），MAE 2.0 BPM——和医生最佳引导（1.9 BPM）几乎一样。这意味着 **DopFone 不需要训练就能用**，是它最关键的"低资源场景可部署"证据。

## 临床医生的访谈：技术能行不等于该这么用

3 位 OB/GYN 访谈被作者主动放进 Results 章节（不是 Discussion 的脚注）——表明作者认真对待"它该不该被用"和"它能不能用"是不同问题：

1. **低资源场景潜力大**（一致同意）：社区健康工作者（CHWs）在偏远村庄、家访高风险孕妇时可立即派上用场，无需多普勒+耦合凝胶+电池供电
2. **家用监测的隐忧**（P1 直接说）："**自我管理对家用胎心仪是关键问题——我们不向任何患者推荐家用多普勒**" — 患者无法解读异常数据可能加剧焦虑或导致不当自处
3. **临床责任问题**（P3）："你仍然需要 MD 实时解读监测数据并给行动建议" — 工具应当是**测量设备**而非建议设备
4. **伦理困境**（P2）："如果在家用这个设备检测到异常但患者无法及时到 ER，责任问题就来了"

> ⚠️ 这部分访谈给出**未解决的开放问题**——技术上 DopFone 已经是"可用"的，但**临床部署架构（谁负责解读？误诊责任谁担？）远未解决**。这种"技术成熟先于制度成熟"的张力是可穿戴 / 数字医疗的常见问题。

## 这意味着什么

1. **第一个真正零硬件的 commodity smartphone FHR 系统**：先前所有"smartphone fetal monitor"要么是带专用探头的（Porter HeraBEAT $$$）、要么是带专用腰带的（Mhajna Invu）、要么是没有临床验证的应用商店软件。DopFone 是第一个用**完全未修改的智能手机**就能跑、有临床数据验证、达到 ±8 BPM 临床阈值的方案
2. **18 kHz 是优雅的工程权衡的 case study**：穿透 vs 分辨率的 trade-off 直接映射到产品形态——比临床 1–3 MHz 慢，但任何手机都能跑
3. **位置无关性 + 用户无需训练 = 真正的部署友好**：CHWs / 助产士 / 孕妇本人都能用
4. **方法学**：把"hardware-defined"的多普勒探针变成"software-defined"的声学传感器，与 Wang 2021 的智能音箱测心率、Nandakumar 2015 的智能手机声呐睡眠呼吸检测同属"主动声学健康感知"新方向

## 局限与作者承认的边界

- **仅单胎妊娠**：双胎妊娠的两个心率叠加在 18 kHz 分辨率下分不开（这正是临床多普勒用 1–3 MHz 的原因）
- **仅 iPhone 15 做临床测试**：跨手机测试是 in vitro 明胶模型——和真孕妇可能有差异
- **仅 2/3 trimester**：1st trimester 胎心更小、位置不稳，未测；这反而是诊断早期问题最有价值的窗口
- **无异常 FHR 样本**：伦理上无法在临床急诊场景部署原型设备，所以训练集都是 110–160 BPM 正常范围；模型在异常 FHR 上的行为未验证
- **N=23 + LOOCV 是初步性的**：作者明确说部署到现实需要"数百用户"扩大数据集

## 技术细节（按需深入）

### 信号处理具体参数
- **Butterworth 带通**：4 阶，中心 18 kHz，带宽 1 kHz（FHR 频偏 < 3.5 Hz，1 kHz 已足够覆盖）
- **Spectrogram**：FFT 窗 256 / 跳跃 64 / Blackman 窗
- **频率重新分配**：基于相位谱信息锐化幅度谱（Cornell ReassignFFT 实现）
- **Novelty function**：相邻帧的频谱差求和，正向取值（裁负），高斯平滑去噪
- **Peak detection**：动态阈值 + 最小间隔约束（依据生理 FHR 110–160 BPM）
- **FHR 计算**：FHR = 60/Δt，10 秒滑动窗口 + 90% 重叠求平均

### ML 特征工程要点
- **MFCC 8 系数**（不是惯用 13）：在 8 系数处效率与判别力达到平衡
- **PLP Tempo with physiological prior**：以 110–160 BPM 均匀分布作先验，避免节拍跟踪算法陷入低频谐波
- **特征丢弃**：HRV 系列特征被实验剔除（孕后期 5–25 BPM 的胎心加减速会污染回归）

## ⚠️ 矛盾与未解决问题

- **临床部署架构未解决**：3 位 OB/GYN 一致认为技术上可用但**家用部署谁负责解读 / 异常时谁担责未定**——技术成熟与制度成熟的时间差
- **跨手机临床验证缺失**：In vitro 明胶模型说"主流手机基本一致"，但**没在真孕妇上验证非 iPhone 15 的精度**——明胶 phantom 不能替代真实组织
- **第一孕期空白**：FHR 异常筛查最重要的早期窗口（< 13 周）未测，且因胎儿小、位置不稳，未来扩展面临真实物理限制

## 🔗 关联

### ECE284 内的方法学谱系
- [[Bhamla_2017_Paperfuge]] — 都是"零附加硬件 / 极低成本健康技术"哲学（纸离心机 / 手机多普勒）；都明确瞄准 low-resource settings；DopFone 还有"零训练成本部署"的更高加成
- [[Song_2024_SmartphoneMicroscope]] — 都是"smartphone-as-medical-device"，但 Song 用相机做血细胞计数（视觉），DopFone 用扬声器+麦克风做胎心率（主动声学）——两个互补的智能手机健康感知模态
- [[Zhang_2015_TROIKA]] — 同为无创心率测量；TROIKA 用 PPG（光学），DopFone 用 Doppler（声学）——不同物理原理但相似挑战（运动伪影、个体差异）。DopFone 是真正第一个把"消费设备 + ML + 临床精度"做到孕产领域的工作
- [[Perez_2019_AppleHeartStudy]] — 都是消费设备进入诊断；都需要应对消费设备硬件多样性（DopFone 用 in vitro 跨手机验证 ≈ Apple Heart Study 把研究锁在一种 Apple Watch 型号上）
- [[Shah_2025_LossOfPulse]] — 都是消费设备 + ML 临床检测；DopFone 强调跨手机一致性（device-agnostic 设计），LossOfPulse 强调极高特异性（用户体验导向）——两条不同的"消费医疗 ML"设计哲学
- [[Luo_2026_NormWear]] — DopFone 用手工特征 + AdaBoost；NormWear 走基础模型 + 自监督。DopFone 是"传统 ML 范式在小样本临床场景"的范例，NormWear 是"基础模型范式在大样本通用场景"的范例
- [[Mason_2024_TemPredict]] — 都是消费级设备进入新临床指标（FHR / 体温-抑郁）；DopFone N=23 临床医生密切监督，TemPredict N=20K 用户自我报告——两端的样本-控制 trade-off

### 跨主题
- [[综合_医疗技术中的种族偏见]] — DopFone 在 BMI ≥ 30 群体上 MAE 升高 33%（1.8 → 2.4 BPM），归因于脂肪组织对 18 kHz 信号的衰减——这暗示**身体组成差异分布**（往往与族群相关）可能成为新的算法公平性挑战源；论文未做种族子群分析，待补
