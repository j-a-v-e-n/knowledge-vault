# 超级个体 — 工具与杠杆

> "拼命是拼不过杠杆的"——视频里这句话是对的。但**杠杆 ≠ 切到新工具**。真正的杠杆是「在保留积累的前提下，把成本结构换掉」。

---

## 起因

Javen 2026-04-29 凌晨看到一段视频文案（[[../raw/web-research/2026-04-29_QClaw_超级个体视频文案]]），核心论点：

1. **技能不转资产 = 廉价劳动力租赁**——可能被时代淘汰
2. **超级个体 = 拥有数字资产 + 独立生产力工具 + 商业逻辑的人**
3. **工具推荐**：腾讯 QClaw（"AI 龙虾"），4000 万 token / 天免费 + multi-agent + 微信扫码 + 安全沙箱

视频引发 Javen 思考——震撼于"4000 万 token 这么多" + "感觉很便宜"，怀疑 Anthropic 订阅是否性价比太低。

## 视频认知层 vs 工具层 — 拆开评估

视频混合了**两件不同的事**：

### 认知层：超级个体 / 数字资产论 ✅

这一层**论点是对的**。跟 [[AI 团队设计原则]] 互补。

| 视频说的 | 翻译成 vault 已有的实践 |
|---|---|
| "把技能沉淀成数字资产" | 你 vault 本身（second brain）= 数字资产 |
| "一个人变成一支队伍" | daemon + skills + approvals.md = 自动化员工 |
| "不要被琐事淹没" | task-board.md + 4 个 skill = 琐事自动化 |
| "学会掌控系统而不是重复劳动" | 你正在做的事情就是这个 |

**Javen 已经在做超级个体的事**。视频不是给你"启示"，而是**确认**你方向对。

### 工具层：QClaw 适不适合？— 拆解事实

| 维度 | QClaw 的 marketing | 现实事实 |
|---|---|---|
| "本地 OpenClaw 龙虾" | 暗示 = Claude Code 中国版 | ❌ 不接 Claude，接 Kimi-K2.5 / Minimax-M2.5 / GLM-5 / DeepSeek-V3.2 |
| "4000 万 token / 天免费" | 听着海量 | ✅ 真的，按市场价值 ~$700/天 |
| "Multi-Agent 并行处理" | 暗示 = Anthropic 三准则的 multi-agent | ⚠️ demo 里是 4 个独立 single-agent 角色（前端/后端/测试），无 agent-to-agent 协作。包装话术 |
| "首个集成安全沙箱" | 防 prompt injection | ⚠️ 真问题但不是 QClaw 独有，主流 agent 工具都在做沙箱 |
| "微信扫码远程操控" | 手机端入口 | ✅ 中国生态完美适配；美国生态错配（你不用微信日常生活）|
| "腾讯文档/会议/邮箱集成" | 办公协作闭环 | ⚠️ 你用 Google Workspace，Anthropic 官方 Gmail/Calendar/Drive MCP 已接 |

## 真正的杠杆 — Claude Code + Router + 多模型路由

视频暴露的真问题：**Anthropic 直接订阅成本高**。

但解法**不是切到 QClaw**，而是利用一个事实——**Claude Code shell 可以接任意 LLM**：

```
Claude Code (UI / 工作流 / vault 集成) 
    ↓ 路由
[claude-code-router] 或 [LiteLLM proxy]
    ↓ 分流到
├── Anthropic Claude（重要任务，质量优先）
├── DeepSeek-V3.2（凌晨 daemon / 重复批处理，便宜 50x）
├── OpenRouter（按需访问 Kimi / GLM / 任何模型）
└── Ollama 本地（完全免费，弱模型，敏感任务）
```

**节省幅度**：典型 50-99%（来源：[Claude Code Router 官方 README](https://github.com/musistudio/claude-code-router) + [OpenRouter integration docs](https://openrouter.ai/docs/guides/coding-agents/claude-code-integration)）。

### 适合 Javen 的路由策略 v0（待实测调整）

| 任务类型 | 推荐模型 | 理由 |
|---|---|---|
| 简历 / cover letter / 重要写作 | Claude Sonnet 4.7 | 质量天花板，用错一次的代价 >> 节省的钱 |
| 主对话日常（修 bug / 整理 / 解释概念）| Claude Sonnet 4.7 或 Sonnet 4.5 | 当前默认，先不动 |
| daemon 凌晨任务（看板推进 / ingest）| DeepSeek-V3.2 | 重复性高，DeepSeek 在 coding / reasoning 评测上接近 Claude 但成本 ~1/50 |
| ai-watch / email-triage 早报 | Kimi-K2.5 或 GLM-5 | 中等任务，长 context（看一堆邮件 / 网页）需要，便宜 |
| 简单查询 / Bash 命令生成 | Ollama 本地（如 Llama 3.3 70B） | 完全免费，离线可用 |

> 这个策略是 v0——具体路由阈值（多少 token 触发哪个模型 / 哪类 prompt 走哪条）需要部署后实测调整。

## 行动建议（Javen 决策清单）

### 不做（避免）

- ❌ **不要弃用 Claude Code 切到 QClaw** — 你过去半个月的 vault / daemon / skills / approvals 是真资产，不能丢
- ❌ **不要在 daemon 上线后第一周折腾路由** — daemon 刚跑顺，先稳定一周再优化成本

### 可以做（双轨并行）

| 行动 | 成本 | 价值 |
|---|---|---|
| **A. 装 QClaw 玩玩** | 0（免费 + 不影响现有）| 看一手腾讯生态 / multi-agent UX / 安全沙箱实现 |
| **B. 调研 claude-code-router** | 0（开源） | 为以后切多模型路由做准备 |
| **C. 部署 claude-code-router 路由 daemon** | 30-60 min 配置 + DeepSeek API key | 凌晨 task 成本降 ~50x；主对话不受影响 |
| **D. 做一份「2026 国内大模型价格横评」笔记** | 1-2h | 知识沉淀；找实习面 ML infra 类岗位有谈资 |

**优先级建议**：
- 这周（4/29-5/3）daemon 稳定 / 投递推进 → 不动
- 下周（5/4-5/10）daemon 跑顺 + 期末缓冲期 → 试 A + B（低成本探索）
- 五月底课业过去 → 考虑 C（真实部署路由）
