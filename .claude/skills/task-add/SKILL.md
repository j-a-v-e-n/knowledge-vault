# 添加新任务到看板

## 工作流

### 1. 读看板找下一个 ID

`Read MyBrain/automation/queue/task-board.md`，扫描所有 `task-NNN`，找最大 N，新任务 ID = `task-{N+1}`（三位数，前导零）。

### 2. 从用户拿信息

如果用户提供的 `$task-title` 已经包含足够信息（明确目标 + 可验证 DoD），不用反复问。**模糊时只问关键的 1–2 个问题**，不要逐字段拷打。

需要的字段（按问的优先级排）：

| 字段 | 默认值 | 何时必须问 |
|---|---|---|
| **标题** | $task-title | 用户没给 |
| **目标**（一句话为什么做） | （问） | 标题不能直接看出动机 |
| **DoD**（怎么算完成） | （问，但简单任务可推断） | 标题动作不可量化时 |
| **优先级** | P1 | 用户暗示有 deadline 或紧急感 |
| **owner** | 看任务性质：写代码/查资料/整理→@claude；选方案/沟通/决策→@javen | 任务两边都有时（拆子任务） |

提问示例：**"为什么做这个？做到什么程度算完成？"**——一次性问完两个最关键的，不要一条一条来。

### 3. 路由判断（参考 automation/CLAUDE.md）

拿到信息后判断：

- **是否拆子任务**：≥3 个独立步骤、或跨多个文件 → 拆；简单任务一行就够
- **owner 是谁**：写代码改文件查资料类→@claude；选方案/批准/沟通类→@javen；混合→拆子任务分别标
- **优先级**：有 deadline ≤7 天 → P0；不确定 → P1；长期备选 → P2

如果任务可以立刻开始（owner=@claude、无阻塞），可建议 Javen 直接挪到"🚧 进行中"——但**默认先放"📥 待启动"**让 Javen 主动启动。

### 4. 写入看板

用 Edit 工具，**精准插入**到"📥 待启动"列末尾（不重写整个文件）。模板：

```markdown
- [ ] **task-{NNN}** | {标题} | #P{0/1/2} | owner: @{claude/javen}
  - **目标**：{一句话}
  - **Definition of Done**：{可验证标准}
  - **创建**：{今天的日期 YYYY-MM-DD}
  - **更新**：{今天的日期}
  - **子任务**：
    - [ ] a. {子任务1}
    - [ ] b. {子任务2}
```

简单任务可省略子任务部分。

### 5. 更新顶部统计

修改文件顶部"**当前状态**"行（待启动 +1）和"**最后更新**"日期。

### 6. 报告

告诉 Javen：

```
✅ 已添加 task-{NNN} 到"📥 待启动"
   {标题} | #P{X} | owner: @{claude/javen}

下一步建议：
- {如果 owner=@claude 且无阻塞，建议立刻挪到进行中}
- {如果有不确定的部分，明确指出 blocked on 什么}
```

## 边界情况

- **同一个任务用户反复说**：先 `/task-check` 看是否已存在，避免重复
- **任务太大**（动辄子任务 10+ 项）：建议 Javen 拆成多个独立 task-NNN，每个 5–8 个子任务
- **看板没有"📥 待启动"列**（结构坏了）：报错并停止，不修复

## 不做的事

- 不擅自添加 Javen 没提的任务（除非用户在主对话里说"再加一个"）
- 不直接放进"🚧 进行中"，除非用户明说"立刻开始"
- 不询问超过 2 个字段——简单任务直接添加，复杂任务问最关键的
- 不在写入前询问"你确定吗"——用户喊 `/task-add` 就是确定要加
