# 任务看板状态检查

读取 `MyBrain/automation/queue/task-board.md`，按以下格式汇报。汇报要简洁，**关注状态和下一步，不要复述任务全文**。

## 汇报模板

```
📊 看板总览（更新于 YYYY-MM-DD）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
🚧 进行中：N 个 | 🔒 阻塞：M 个 | 📥 待启动：K 个 | ✅ 本周完成：J 个

━━━ 🚧 进行中 ━━━
• [task-001] 标题 (P0, @claude) — 子任务进度 X/Y
   下一步：[next subtask description]

━━━ 🔒 阻塞 ━━━
• [task-007] 标题 (P1, @claude) — ⚠️ 等 @javen 决定 X 还是 Y
   建议：[一句话解阻塞建议]

━━━ 📥 待启动（按优先级排）━━━
• [P0] task-008 — 一句话标题
• [P1] task-002 — 一句话标题

━━━ 💡 给 Javen 的建议 ━━━
1. 先解 task-007 的阻塞（30 秒决定一下）
2. 我可以立刻推 task-001 的子任务 d / task-003 的子任务 a
3. P0 的 task-008 该开始了，要我接管吗？
```

## 实施步骤

1. **Read** `MyBrain/automation/queue/task-board.md`
2. 解析四列（📥 待启动 / 🚧 进行中 / 🔒 阻塞 / ✅ 已完成）
3. 对每个任务提取：ID、标题、优先级、owner、子任务进度（已完成/总数）、阻塞原因
4. 按上面模板汇报
5. 在"💡 给 Javen 的建议"里明确：
   - 哪些阻塞最容易解（只需要 Javen 一句话）
   - 我（Claude）现在能立刻推进的事
   - 是否有 P0 待启动任务还没人接

## 边界情况

- **看板为空**：明确说"看板里现在没有任务，要不要 `/task-add` 一条？"
- **看板有解析错误**（格式坏了）：报错并指出第几行，建议 Javen 修复，**不要尝试修复**
- **本周完成数**：扫"✅ 已完成"列里 done 日期在过去 7 天内的任务

## 不做的事

- 不修改看板（这个 skill 是只读的）
- 不展开每个任务的全部子任务列表（只报"X/Y 完成 + 下一步是什么"）
- 不评判任务好坏，只报状态
